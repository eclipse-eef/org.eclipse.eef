HTML Renderer

Description

The HTML Renderer widget is used to display values that can be represented as HTML Content. On top of the common properties of all widgets (identifier, labelExpression, helpExpression, isEnabledExpression, propertyValidationRules), it has the following properties:

  • valueExpression: This expression is used to compute the value that can be converted to HTML using toHtmlExpression. The computed result of the valueExpression is a String which is not necessarily in HTML format. This expression must return a string.
  • editExpression: This expression will be executed when the user will click on the edit button. This expression is executed if an Operation is provided under the widget. This expression does not have to return anything.
  • toHtmlExpression: This mandatory expression is used to compute the HTML content to display. It transforms the result of the valueExpression (stored in newValue variable) into HTML content. The computed result is a String in HTML format. This expression must return a string.
  • editionToolExpression : This optional expression is used to open a tool to edit the original content (see org.eclipse.eef.ide.ui.ext.widgets.markup.markdown.MarkdownService#editMarkdownExpression(String, boolean) for an example to edit Markdown content). A newValue variable is available containing the result of valueExpression. If this expression is not provided then the Edit Button is not displayed. This expression should return a String that is then given to the editExpression to execute an action with the result.
  • numberOfLine : The number of lines to be displayed, which determines the height of the HTML Renderer widget. Be aware that this is an estimation. Indeed the text size modification done by CSS styling is not used here.
  • cssExpression: This optional expression is used to provide some CSS content. If this expression is used, the toHtmlExpression should only return the content of a body.
